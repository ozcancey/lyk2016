Sürüm Kontrol Sistemi

Sürüm Kontrol Nedir?

Sürüm kontrol, dosya ya da dosyalar üzerinde yapılan değişiklikleri kayıt altına alan, ve bu değişiklikler arasında geçiş yapmanızı sağlayan bir sistemdir.

GIT Nedir?

GIT, dağıtık çalışan bir sürüm kontrol sistemidir.

Tarihçe

Linus Torvalds tarafından 2005 yılında yayınlanmıştır.

Dağıtık?

  • Geliştiriciler tüm geçmişe sahiptir.
  • Çevirimiçi olmayı gerektirmez.
  • Değişiklikler sunucu aracılığı ile paylaşılabilir.

Başlangıç

mkdir yeniproje
cd yeniproje
git init
touch README.md
git add .
git commit -m "İlk commit!"

Temel Komutlar

git config

Kullanıcı adı, e-posta vb. temel ayarları yapmayı sağlayan komuttur. 

git config --global user.name "M. Onur Çelik"
git config --global user.email monrcelik@gmail.com

git init

Bulunan dizini, sürüm kontrolü altına alır. 

git init

git clone

Var olan bir git repository'sini klonlar. 

git clone https://github.com/git/git.git

git add

Varolan dosyaları commit için hazırlar. Yeni oluşturulan dosyaları sürüm kontrolüne ekler.

Belli bir dosya için;

git add README.md

Bulunulan dizin için;

git add .

Projedeki tüm dosyalar için;

git add *

git status

Dosyaların durumunu bildirir. 

git status

git commit

Eklenmiş değişiklikleri kaydeder. 

git commit -m "Commit mesajı."

git push

Kaydedilmiş değişiklikleri uzak sunucuya gönderir. 

git push origin master

git pull

Uzak sunucudaki değişiklikleri yerel sunucuya aktarır. 

git pull origin master

git log

Repository geçmişini gösterir. 

git log
git log --oneline
